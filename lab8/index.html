ใบงานที่ 8: การบันทึกสถิติมิติตามเวลาด้วย InfluxDB (Data Logging)

หลักสูตรการพัฒนาระบบ Hybrid Local/Cloud IoT Node ด้วย ESP32 / ESP8266

ระบบส่งใบงานปฏิบัติการออนไลน์
คู่มือปฏิบัติการทดลอง

วัตถุประสงค์

  1. เพื่อเข้าใจโครงสร้างฐานข้อมูลอนุกรมเวลา (Time-Series Database)
  2. เพื่อบันทึกข้อมูลภูมิอากาศและสถานะรีเลย์จาก ESP32 ลงใน InfluxDB โดยใช้ Node-RED

เครื่องมือที่ใช้ในการทดลอง

ชื่อซอฟต์แวร์ / ไลบรารี คำสั่งรัน / ติดตั้ง
InfluxDB (Docker) docker run -d -p 8086:8086 influxdb
node-red-contrib-influxdb ติดตั้งใน Node-RED ผ่าน Palette Manager

โครงสร้างข้อมูล InfluxDB (Data Schema)

ประเภทโครงสร้าง ชื่อคีย์ รายละเอียด
Bucket esp32_iot_data ชื่อถังเก็บฐานข้อมูล
Measurement climate_metrics ตารางเก็บข้อมูลเปรียบเสมือน Table
Tags device_id = ESP32-ClimateNode คีย์สำหรับแยกแยะแหล่งที่มา (Index)
Fields temp, humidity, analog ข้อมูลค่าตัวเลขดิบที่ต้องการจัดเก็บ

ขั้นตอนการทดลอง

  1. เปิด InfluxDB UI ผ่านเว็บเบราว์เซอร์ที่พอร์ต http://localhost:8086 สมัครเข้าใช้งานครั้งแรกเพื่อรับ API Token, สร้าง Organization และสร้าง Bucket ชื่อ esp32_iot_data
  2. ใน Node-RED ให้ติดตั้งไลบรารี node-red-contrib-influxdb
  3. ลากโหนด influxdb out มาวาง และตั้งค่าการเชื่อมโยงข้อมูลดังนี้:
    • Version: 2.0 (หรือ 1.8 ตามเวอร์ชันที่ใช้งานจริง)
    • URL: http://localhost:8086
    • Token / Org / Bucket: กรอกข้อมูลที่คัดลอกมาจาก InfluxDB UI
    • Measurement: climate_metrics
  4. ลากโหนด function มาเชื่อมต่อก่อนหน้าโหนด InfluxDB เขียนสคริปต์เพื่อจัดรูปแบบ Payload ให้อยู่ในโครงสร้าง Field Key ดังต่อไปนี้:
JavaScript Payload Formatter 
// ดึงค่าตัวเลขทศนิยมจากข้อมูลเซ็นเซอร์ที่ได้รับ
const val = parseFloat(msg.payload);

// จัดรูปแบบเป็น Field key และส่งออกไป
msg.payload = {
    temp: val
};
return msg;

โจทย์ท้าทายการทดลอง (Challenge)

โจทย์: เขียน Flow เพิ่มเติมเพื่อบันทึกสถานะเปิด-ปิดของรีเลย์พัดลมและปั๊มน้ำ (พารามิเตอร์ข้อความ "ON" หรือ "OFF" แปลงให้เป็นตัวเลขจำนวนเต็ม 1 หรือ 0) ลงไปใน Measurement เดียวกัน เพื่อใช้สำหรับการคำนวณและประเมินชั่วโมงการทำงานของฮาร์ดแวร์ในอนาคต
