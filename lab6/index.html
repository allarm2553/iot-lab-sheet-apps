ใบงานที่ 6: การบูรณาการโครงงาน Hybrid Node ขั้นสูง

(The Grand Project Challenge) - หลักสูตรการพัฒนาระบบ Hybrid Local/Cloud IoT Node ด้วย ESP32 / ESP8266

ระบบส่งใบงานปฏิบัติการออนไลน์
คู่มือข้อกำหนดโครงการบูรณาการ

วัตถุประสงค์

  1. เพื่อบูรณาการระบบควบคุมแบบเรียลไทม์ภายในแลนวงเดียวกัน (WebSockets) และระบบการรายงานข้อมูลระดับโลก (Cloud MQTT) ให้อยู่ร่วมกันในแพลตฟอร์ม ESP32 / ESP8266 ตัวเดียว
  2. เพื่อสร้างระบบการทำงานที่ทนทานต่อข้อผิดพลาด (Fault-Tolerant) สามารถรับมือและทำงานต่อไปได้ปกติแม้อุปกรณ์จะตัดขาดจากเครือข่ายอินเทอร์เน็ตภายนอก (Offline State)
  3. เพื่อออกแบบและพัฒนา Local Web Dashboard ที่มีกราฟิก Real-time แสดงสถานะอุปกรณ์ ตอบสนองกับ WebSocket JSON payload จาก ESP32 โดยตรง

โจทย์โครงงานบูรณาการร่วม (The Grand Project Challenge)

ให้นักเรียนใช้ทักษะความรู้และไลบรารีทั้งหมดที่ศึกษาผ่านบทเรียนใน ใบงานที่ 1 ถึง 5 นำมารวมโค้ดโครงการใหญ่ชิ้นเดียวให้ทำงานสอดคล้องกัน ทั้งฝั่ง Firmware (.ino) และ Web Dashboard (index.html + styles.css) โดยโครงงานต้องมีระบบการตอบสนองที่ระบุคุณสมบัติดังนี้:

  1. การทำงานฝั่งกายภาพ (Hardware Layer):
    • คอยอ่านค่าจากเซ็นเซอร์ DHT11 (GPIO 33) เพื่อรับค่าอุณหภูมิและความชื้น และแปลงแรงดันอะนาล็อกจากขาพิน GPIO 34 (หรือ A0 สำหรับ ESP8266) เพื่อใช้เป็นค่าความชื้นดิน (Soil Moisture) แสดงผลทุกค่าผ่านหน้าจอ OLED อย่างสม่ำเสมอ
    • ควบคุมรีเลย์พัดลมระบายอากาศ (GPIO 13) และปั๊มน้ำพ่นหมอก (GPIO 14) ด้วยตรรกะ Hysteresis อัตโนมัติ และรับคำสั่ง Override จาก Dashboard ได้
  2. การโฮสต์ Web Dashboard ภายในเครือข่าย (Local Web Server):
    • โฮสต์ไฟล์ index.html และ styles.css จาก LittleFS ผ่าน Web Server พอร์ต 80
    • เปิด WebSocket Server บนพอร์ต 81 เพื่อส่ง JSON payload ไปยัง Dashboard ทุก 2 วินาที โดย payload ต้องมีฟิลด์ครบ: {"temp": 29.5, "humidity": 65.0, "soil": 38.0, "fan": true, "mist": false, "threshold": 30.0, "mqtt": true}
    • Dashboard ต้องแสดง Connection Status Bar 4 indicators (WiFi, Local WS, MQTT Broker, ESP32) อัปเดตตาม WebSocket state อัตโนมัติ
    • Dashboard ต้องมีกราฟิก SVG แบบ Animated สำหรับพัดลม (หมุนเมื่อ ON) และปั๊มน้ำพ่นหมอก (หยดน้ำลอยขึ้นเมื่อ ON) ที่ตอบสนองกับ WebSocket payload
  3. สถาปัตยกรรม MQTT และโครงสร้าง Topic (Hybrid Cloud):
    • เชื่อมต่อ MQTT Broker (broker.emqx.io:1883) แบบ Non-Blocking Reconnection ทุก 10 วินาทีด้วย millis() (ห้ามใช้ while หรือ delay())
    • ใช้โครงสร้าง MQTT Topic แยกตามฟังก์ชัน โดย prefix ทั้งหมดต้องเป็น esp32-climate-node/ ดังนี้:
    ประเภท Topic Path รายละเอียด
    Subscribe esp32-climate-node/switch/cooling_fan_relay/command รับคำสั่งเปิด/ปิดพัดลม (ON/OFF)
    Subscribe esp32-climate-node/switch/mist_pump_relay/command รับคำสั่งเปิด/ปิดปั๊มน้ำ (ON/OFF)
    Subscribe esp32-climate-node/climate/temp_threshold/set ปรับค่าเกณฑ์อุณหภูมิจากระยะไกล
    Publish esp32-climate-node/sensor/room_temperature/state ค่าอุณหภูมิ (°C)
    Publish esp32-climate-node/sensor/room_humidity/state ค่าความชื้นสัมพัทธ์ (%)
    Publish esp32-climate-node/sensor/analog/state ค่าความชื้นดิน (%)
    Publish esp32-climate-node/switch/cooling_fan_relay/state สถานะพัดลม (ON/OFF)
    Publish esp32-climate-node/switch/mist_pump_relay/state สถานะปั๊มน้ำ (ON/OFF)
  4. ลอจิก Hysteresis การควบคุมอัตโนมัติ:
    • พัดลม: เปิด (ON) เมื่ออุณหภูมิ > tempThreshold  |  ปิด (OFF) เมื่ออุณหภูมิ < tempThreshold - 0.5
    • ปั๊มน้ำพ่นหมอก: เปิด (ON) เมื่อความชื้นดิน < 40%  |  ปิด (OFF) เมื่อความชื้นดิน > 50%
    • เมื่อผู้ใช้ Override ผ่าน Dashboard หรือ MQTT ต้องส่ง state feedback กลับผ่าน Publish topic ทันที

ข้อกำหนดทางเทคนิคที่ห้ามฝ่าฝืน

  • ฟังก์ชัน MQTT Reconnection ต้องเขียนแบบ Non-Blocking โดยใช้ millis() ตรวจสอบช่วงเวลา — ห้ามใช้ while(!connected) หรือ delay() ภายในลูปเด็ดขาด เพื่อป้องกัน WebSocket หยุดทำงาน
  • WebSocket JSON payload ต้องมีฟิลด์ "mqtt": true/false เพื่อให้ Connection Status Bar บน Dashboard แสดงสถานะ MQTT Broker ได้อัตโนมัติ
  • Dashboard ต้องโหลด CSS จาก styles.css ภายนอก (ห้ามใช้ inline <style> ทั้งหมด) และไฟล์ทั้งสองต้องอัปโหลดไปยัง LittleFS พร้อมกัน
