ใบงานที่ 9: การวิเคราะห์และแสดงผลข้อมูลย้อนหลังด้วย Grafana

หลักสูตรการพัฒนาระบบ Hybrid Local/Cloud IoT Node ด้วย ESP32 / ESP8266

ระบบส่งใบงานปฏิบัติการออนไลน์
คู่มือปฏิบัติการทดลอง

วัตถุประสงค์

  1. เพื่อเชื่อมต่อระบบ Grafana เข้ากับฐานข้อมูล InfluxDB
  2. เพื่อออกแบบหน้าแดชบอร์ดระดับสูง วิเคราะห์เปรียบเทียบข้อมูลเซ็นเซอร์และสถานะอุปกรณ์ย้อนหลัง

เครื่องมือที่ใช้ในการทดลอง

ชื่อซอฟต์แวร์ คำสั่งรันระบบผ่าน Docker
Grafana docker run -d -p 3000:3000 grafana/grafana

ขั้นตอนการทดลอง

  1. เปิดหน้าเว็บ Grafana ที่ URL: http://localhost:3000 ลงชื่อเข้าใช้งานด้วยสิทธิ์ผู้ดูแลระบบเริ่มต้น (Admin Credentials: admin / admin)
  2. การตั้งค่า Data Source (แหล่งข้อมูล):
    • ไปที่ Connections → Data Sources → คลิกปุ่ม Add data source
    • เลือกประเภทแหล่งข้อมูลเป็น InfluxDB
    • เลือก Query Language เป็นแบบ Flux (สำหรับฐานข้อมูล InfluxDB v2.x)
    • ระบุ URL เชื่อมต่อ: http://localhost:8086 (หรือไอพีเซิร์ฟเวอร์ฐานข้อมูลหากคนละเครื่องกัน)
    • กรอกข้อมูล Organization, Bucket และ API Token ที่ได้จาก InfluxDB จากนั้นกดปุ่ม Save & test เพื่อยืนยันการเชื่อมต่อ
  3. การสร้างแดชบอร์ดและแผนภูมิ Panel:
    • สร้างแดชบอร์ดใหม่ (Create Dashboard) และกดเลือก Add a new panel
    • ระบุคำสั่งในการดึงข้อมูลแบบ Flux Query เพื่อแสดงผลค่าอุณหภูมิย้อนหลังดังสคริปต์นี้:
Flux Query (Temperature Field) 
from(bucket: "esp32_iot_data")
  |> range(start: v.timeRangeStart, stop: v.timeRangeStop)
  |> filter(fn: (r) => r["_measurement"] == "climate_metrics")
  |> filter(fn: (r) => r["_field"] == "temp")
  |> aggregateWindow(every: v.windowPeriod, fn: mean, createEmpty: false)
  |> yield(name: "mean")
  1. ปรับเปลี่ยนประเภทแผงแสดงผล (Panel Options) ด้านขวาให้เป็นกราฟความต่อเนื่องของเวลาแบบ Time Series และกำหนดสีของกราฟให้เป็นโทนสีน้ำเงินสว่างสไตล์นีออน (Neon Blue) เพื่อให้เห็นเส้นกราฟที่ชัดเจน
  2. กดบันทึกความคืบหน้า (Save Dashboard) ตั้งชื่อระบบว่า "ESP32 Advanced Climate Dashboard"

โจทย์ท้าทายการทดลอง (Challenge)

โจทย์: สร้างแผงแสดงผลประเภท Stat Panel (การ์ดแสดงผลตัวเลขเดี่ยวขนาดใหญ่) แสดงข้อมูลอัตราเฉลี่ยความชื้นในดิน (Analog Sensor) ย้อนหลัง 24 ชั่วโมงที่ผ่านมา พร้อมตั้งเกณฑ์ขีดจำกัด (Threshold) ว่าหากค่าความชื้นดินเฉลี่ยต่ำกว่า 30% ให้ระบบเปลี่ยนสีตัวเลขผลลัพธ์เป็นสีแดงเพื่อเตือนภัยโดยอัตโนมัติ
